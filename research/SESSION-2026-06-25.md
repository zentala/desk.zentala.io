## What we established about the project

### Product definition (finally clear)

Open Smart Desk v2 is a **nudge system**, not automation:

- Hardware: **RP2040-Tiny** (~20 PLN) + **VL53L0X** laser sensor (~20 PLN) → USB to PC
- Detects **sitting vs. standing**: desk height from laser (low = sitting, high = standing)
- Detects **presence at computer**: mouse + keyboard OS events (no PIR needed)
- App: **Electron in system tray**
- Output: **OS-native notifications** — non-annoying, context-aware
- User **manually presses desk button** — system does NOT auto-raise
- Gamification: **Duolingo-style** — streaks, XP, adaptive timing, positive reinforcement

### What exists

- Working prototype (used by creator ~50% of working time, not connected to every computer)
- Loose experimental codebase (Electron + laser sensor + mouse tracking) — unstable
- Research: `research/user-quotes.xml` (26 real quotes from real sources)
- Spec: `research/hardware-v2-spec.md`

### Decision: rewrite from scratch

The existing codebase was exploratory experimentation. Decision: rewrite cleanly as v1 product.

## MVP plan

### What to ship first

**Electron tray app** with:
1. Mouse/keyboard activity tracking (presence detection — no hardware needed)
2. USB serial read from RP2040-Tiny + VL53L0X (optional hardware mode)
3. Single notification: "You've been sitting for X minutes"
4. One config: threshold time (default 45 min)

**What NOT in MVP**: streaks, XP, dashboard, history DB, calendar integration,
gamification, multi-desk support, auto-calibration.

### Release plan

1. Rewrite app cleanly
2. Two modes: software-only (mouse tracking) + hardware (+ height sensor)
3. GitHub release with binaries (Windows + Linux + macOS)
4. Minimal README + screenshot of notification
5. Post to: r/StandingDesk, r/selfhosted, HA Community

### Validation signal

- 20+ GitHub stars in first week → proceed to hardware kit
- 0 stars → change message, not product

## Open questions / decisions needed

### Algorithm (most critical — do this first before coding)

The notification algorithm is the core of the product. Needs to be designed carefully.
Questions to answer:

- [ ] After how many minutes of sitting do we send the first notification?
- [ ] What is "sitting"? (height below threshold AND mouse/keyboard active)
- [ ] What is "away"? (no activity for X minutes → don't count as sitting time)
- [ ] What happens AFTER notification? Wait Y minutes, then repeat? Or wait for standing event?
- [ ] What if user stood for only 2 minutes — does that reset the timer?
- [ ] How to avoid notification during meetings? (detect: no keyboard for 30min but mouse moves = maybe meeting)
- [ ] Night mode / working hours config?

**→ Design algorithm as state machine before writing any code.**

### Hardware questions

- [ ] VL53L0X mounting position: under blat pointing down, or on desk leg?
- [ ] Calibration UX: how does user set sitting/standing height thresholds?
- [ ] USB Serial vs HID — which firmware approach for RP2040-Tiny?
- [ ] PCB design: RP2040-Tiny + VL53L0X on one black PCB
- [ ] Enclosure v2 DXF (much smaller than v1 — no relay board, no 230V)

### Business / community questions

- [ ] Where to find first users? (r/StandingDesk, HA Community, r/selfhosted)
- [ ] Free software + paid hardware kit, or GitHub Sponsors, or both?
- [ ] Hardware kit price point: 150 PLN? 200 PLN?
- [ ] When to add Duolingo-style gamification? (after 100 active users)
